# System Architecture & Design Alignment Report

This report explains exactly what the Agentic AI-Powered Smart DevOps Assistant is, how it works under the hood, and how the codebase aligns perfectly with the academic requirements outlined in your **Methodology and Design (ITBIN-2211-0229)** documentation.

## What is this system?
Modern software engineering relies heavily on DevOps tools (Kubernetes, Docker, GitHub Actions) to deploy code. However, managing these tools is incredibly complex and requires manual intervention when things break.

Your system is an **Agentic AI-Powered DevOps Assistant**. It acts as a virtual, autonomous DevOps Engineer. Instead of a human manually reading terminal logs to figure out why a pod crashed or why a GitHub pipeline failed, the user simply asks the Assistant in plain English. The Assistant automatically retrieves the logs via APIs, analyzes them using an LLM (Groq LLaMA 3.1), and provides the solution or executes the fix.

## How It Works (The 3 Layers)

As per **Section 5.1 (Navigation / Module Structure)** in your PDF, the system is divided into three layers:

### 1. React.js Frontend (User Interface Layer)
This is the visual dashboard built with React and Tailwind CSS.
- **Dashboard & Infrastructure Pages:** These pages fetch real-time Kubernetes data (via the backend) to display CPU/Memory metrics, running pods, and healthy deployments.
- **Pipelines Page:** Fetches workflow data from GitHub Actions.
- **Incidents & Audit Logs:** Displays system events. *(Note: Currently, as per Section 3.2.4 "rapid prototyping", these pages display mock UI data to represent how the final LLM-based Log Analyzer Agent will output its findings).*
- **Assistant Screen:** The core chat interface where the user types natural language queries.

### 2. Node.js Backend (Application Logic Layer)
This is the API server running on Express.js (`apps/api`).
- **DevOps Tool Integration:** It contains adapters (`packages/diagnosis/src/k8s-adapter/real.ts` and `github-adapter`) that securely talk to your local Kubernetes cluster and GitHub repository via REST APIs.
- It acts as the secure middleman. The React frontend never talks to Kubernetes directly; it asks the Node.js backend to fetch the data.

### 3. Agentic AI Core (LLM Orchestration Layer)
This is the brain of the application (`packages/diagnosis/src/analyzer.ts`).
- When a user asks a question in the Assistant Screen, the Node backend forwards it to the Agent layer.
- The AI uses the **Groq LPU API** (fulfilling Section 3.2.1 Technical Feasibility) to process the request rapidly.
- It is designed to interpret pipeline logs and Kubernetes errors and return actionable insights.
