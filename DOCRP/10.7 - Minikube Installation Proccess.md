# Setup, Connection, and Testing Guide for Minikube (Hyper-V)

This document is a complete, battle-tested guide for setting up Minikube on Windows using the Hyper-V driver, linking it to the AAPSD-Assistant, starting all required services, and performing full system verification tests.

---

## Prerequisites Checklist

Before starting, confirm you have all of these:

- [ ] Windows 11 with Hyper-V enabled (requires Pro/Enterprise edition)
- [ ] Docker Desktop installed (needed for running Redis)
- [ ] Node.js v22 or higher installed
- [ ] Git installed
- [ ] At least 6GB of free RAM

---

## Phase 1: Install Minikube

If you need to install or reinstall Minikube from scratch:

1. **Open PowerShell as Administrator** (Right-click → Run as Administrator)
2. **Install Minikube**:

   ```powershell
   winget install Kubernetes.minikube
   ```

   > If this returns "No package found", use the exact ID above including the dot.

3. **Reload environment variables** (so the terminal finds the new command):

   ```powershell
   $env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
   ```

4. **Verify installation**:
   ```powershell
   minikube version
   ```
   ✅ Expected: `minikube version: v1.38.x`

---

## Phase 2: Start the Minikube Cluster

> ⚠️ **IMPORTANT**: This must be run from an **Administrator PowerShell terminal** because Hyper-V requires elevated privileges.

1. **Start the cluster**:

   ```powershell
   minikube start --driver=hyperv
   ```

   _This takes 3-10 minutes the first time. It downloads the VM boot image, creates a virtual machine using Hyper-V, installs Kubernetes inside it, and configures kubectl to point to it._

2. **Verify the cluster is running**:
   ```powershell
   kubectl get nodes
   ```
   ✅ Expected:
   ```
   NAME       STATUS   ROLES           AGE   VERSION
   minikube   Ready    control-plane   1m    v1.35.x
   ```

## Phase 3: Start Redis (Required for Job Queue)

The AAPSD-Assistant uses Redis for the background job queue. This must be running before starting the API.

> ⚠️ **IMPORTANT**: You must open **Docker Desktop** first before running any Docker commands.

### Step 1: Open Docker Desktop

1. Press the **Windows key**, search for "Docker Desktop", and open it.
2. Wait for the Docker Desktop window to show the whale icon is green (running).
3. This usually takes 30-60 seconds.

### Step 2: Start a Redis Container

Once Docker Desktop is running, open a **normal PowerShell terminal** (not Administrator) and run:

```powershell
docker run -d --name aapsd-redis -p 6379:6379 redis:alpine
```

### Step 3: Verify Redis is Running

```powershell
docker ps
```

✅ Expected: You see `aapsd-redis` with status `Up`.

### ⚠️ Troubleshooting: "failed to connect to docker API"

This error means Docker Desktop is not open yet. The fix is simple:

1. Open Docker Desktop from the Start Menu.
2. Wait 30-60 seconds for it to fully start.
3. Run the `docker run` command again.

### Re-starting Redis After a Reboot

If you restart your computer, Redis will stop. To bring it back:

```powershell
docker start aapsd-redis
```

## Phase 4: Generate Kubernetes Credentials for the API

The AAPSD-Assistant needs a secure ServiceAccount token to communicate with Minikube. We have automated this process with a script.

1. **Open a normal PowerShell terminal** and navigate to the project root:

   ```powershell
   cd C:\Users\isuru\Downloads\Codex\AAPSD-Assistant
   ```

2. **Run the connection script**:

   ```powershell
   .\scripts\setup-minikube.ps1
   ```

   _This script automatically creates a ServiceAccount named `aapsd-assistant`, assigns `cluster-admin` rights for testing, and generates a long-lived secret token._

3. **Copy the output values** into your `apps/api/.env` file:
   ```env
   K8S_SKIP_TLS_VERIFY=true           # Required for Minikube's self-signed cert
   K8S_API_SERVER_URL=https://172.26.x.x:8443   # From script output
   K8S_TOKEN=eyJhbGciOiJSUzI1NiIs...           # From script output
   K8S_ALLOWED_NAMESPACES=default,kube-system
   DEPLOYMENT_PROFILE=staging-remote
   ```
   > **Note**: `K8S_SKIP_TLS_VERIFY=true` is required because Minikube uses a self-signed TLS certificate. Set this to `false` when deploying to a real production cluster.

### Test 3: Governed Action Lifecycle (Full End-to-End)

This is the most important test — it verifies the entire request → approve → execute → audit chain.

#### Step A: Deploy a Sample App to Minikube

In an **Administrator PowerShell terminal**, create a test deployment:

```powershell
kubectl create deployment hello-nginx --image=nginx
kubectl get deployments
```

✅ Expected: `hello-nginx` shows `1/1 READY`.

#### Step B: Request a Governed Action

1. In the Web App, navigate to **Governed Actions**.
2. Click **Request New Action**.
3. Fill in:
   - **Action Type**: `kubernetes.deployment.restart`
   - **Deployment Name**: `hello-nginx`
   - **Namespace**: `default`
4. Click **Submit**.
5. ✅ Expected: Status shows `pending`.

#### Step C: Approve the Action

1. Ensure you are logged in as an `administrator` or `approver` role user.
2. Navigate to **Governed Actions**, find the pending plan.
3. Click **Approve**.
4. ✅ Expected: Status changes to `approved`.

#### Step D: Execute the Action

1. Click **Execute** on the approved plan.
2. ✅ Expected: Status changes to `processing` then `completed`.

#### Step E: Verify on Cluster

Confirm the deployment was restarted on the cluster:

```powershell
kubectl rollout status deployment/hello-nginx
```

✅ Expected: `deployment "hello-nginx" successfully rolled out`

## Troubleshooting Reference

| Error                                    | Cause                                     | Fix                                                              |
| ---------------------------------------- | ----------------------------------------- | ---------------------------------------------------------------- |
| `ECONNREFUSED 6379`                      | Redis not running                         | Open Docker Desktop, then run `docker start aapsd-redis`         |
| `failed to connect to docker API`        | Docker Desktop not open                   | Open Docker Desktop and wait 30-60 seconds                       |
| `unable to verify the first certificate` | Minikube self-signed cert                 | Ensure `K8S_SKIP_TLS_VERIFY=true` is in `.env`                   |
| `Invalid encrypted data format`          | Corrupted `.env` file                     | Rewrite the `.env` file to remove invisible null byte characters |
| Migration fails with `null values`       | Old data violates new NOT NULL constraint | Run `npm run db:clean-dirty` then `npm run db:migrate`           |
| `Route GET:/api/v1/config not found`     | Old `dist/` build                         | Run `npm run build` in `apps/api` before `npm start`             |
| `minikube: not recognized`               | PATH not refreshed                        | Close and reopen PowerShell, or reload PATH manually             |
