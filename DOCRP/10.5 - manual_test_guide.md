# AAPSD Assistant - Full Manual Test Guide (Updated with Firebase Auth)

This guide covers setting up, running, and verifying the AAPSD Assistant locally, including the new Firebase Auth and Supabase PostgreSQL backend.

## Prerequisites

1. **Node.js** (v18+)
2. **Git**
3. **A Firebase Project** (to configure `.env` variables if using your own, though defaults are provided in the repo)
4. **Supabase Database** (IPv4 connection pooler URL)

## 1. Environment Setup

### 1.1 Backend Configuration (`apps/api/.env`)

Ensure `apps/api/.env` contains the following variables:

```env
DATABASE_URL="postgresql://postgres.kopjowqxltigwkdxdfja:[YOUR-PASSWORD]@aws-1-ap-northeast-2.pooler.supabase.com:6543/postgres"

# GitHub OAuth App Credentials (for Settings Page Integration)
GITHUB_CLIENT_ID="your_client_id"
GITHUB_CLIENT_SECRET="your_client_secret"

# Firebase Admin Service Account Location
FIREBASE_SERVICE_ACCOUNT_PATH="firebase-adminsdk.json"
```

### 1.2 Firebase Admin SDK Configuration

Create a file named `firebase-adminsdk.json` in the `apps/api/` directory with your Firebase service account credentials.

### 1.3 Database Migration

1. Navigate to the API workspace:
   ```bash
   cd apps/api
   ```
2. Push the schema to the Supabase PostgreSQL database:
   ```bash
   npm run db:push
   ```

## 2. Starting the Application

1. Install all dependencies from the root of the project:

   ```bash
   npm install
   ```

2. Start the Backend API (Terminal 1):

   ```bash
   npm run dev --workspace=@aapsd/api
   ```

   _Expected Output: Logs indicating the server is listening on port 3000._

3. Start the Frontend App (Terminal 2):
   ```bash
   npm run dev --workspace=@aapsd/web
   ```
   _Expected Output: Vite development server starts on port 5173._

## 3. Testing User Flows

### 3.1 Authentication

1. Open `http://localhost:5173` in your browser.
2. You will be redirected to the custom Tailwind-styled Login Page.
3. Test **Email/Password Authentication** or **Google Sign-In**.
4. Upon successful login, you will be redirected to the Dashboard.

### 3.2 Dashboard & WebSocket Streaming

1. Navigate to the **Dashboard**.
2. Wait a few seconds for the WebSockets connection to establish.
3. Verify that the "Live Infrastructure Status" displays active nodes/pods.
4. Verify that "Recent Pipeline Runs" populates.

### 3.3 Integrations & Settings (GitHub OAuth)

1. Navigate to **Settings** from the sidebar.
2. Under "Connected Services", locate the **GitHub** row.
3. Click the **Connect** button.
4. A popup window should open, redirecting you to GitHub's authorization page.
5. Authorize the application. The popup will automatically close and save the token securely to the `user_integrations` database table.
6. Refresh the page or check the Pipelines view to verify data is being fetched using your GitHub OAuth token.

### 3.4 AI Assistant

1. Navigate to the **AI Assistant** tab.
2. Type a query (e.g., "Analyze recent deployment failures").
3. Submit the query and wait for the response from the LLM via OpenRouter.
4. Verify the markdown renders correctly and includes relevant context.

## 4. Troubleshooting

- **Database Connection Fails:** Ensure you are using the IPv4 Transaction Pooler URL (`aws-1-...pooler.supabase.com`) if you are running locally on Windows/Node.js, as Supabase's direct URLs often fail due to IPv6 DNS resolution issues.
- **Firebase Auth Errors:** Check that your `firebase.ts` config on the frontend matches the `firebase-adminsdk.json` on the backend.
- **OAuth Popup Closes but Doesn't Save:** Ensure the frontend URL exactly matches the origin handling the `window.postMessage` logic in `Settings.tsx`.
