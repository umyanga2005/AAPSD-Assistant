# Docker Desktop Kubernetes — Complete Setup, Connection & Testing Guide

This guide covers everything needed to connect the AAPSD-Assistant to Docker Desktop's built-in Kubernetes cluster and run a full system test.

---

## Prerequisites Checklist

Before starting, confirm:

- [ ] Docker Desktop is **open** and the engine is **running** (whale icon in system tray)
- [ ] Docker Desktop **Kubernetes is enabled** (Settings → Kubernetes → Enable Kubernetes ✅)
- [ ] The Kubernetes cluster shows **"Active"** in Docker Desktop
- [ ] Node.js v22+ is installed

---

## Phase 1: Verify Docker Desktop Kubernetes is Active

1. Open **Docker Desktop**
2. Click **Kubernetes** in the left sidebar
3. Confirm the Cluster status shows **Active** and **Kubernetes version** is displayed
4. Open a PowerShell terminal and verify:
   ```powershell
   kubectl get nodes
   ```
   ✅ Expected:
   ```
   NAME             STATUS   ROLES           AGE   VERSION
   docker-desktop   Ready    control-plane   5m    v1.36.x
   ```

---

## Phase 2: Start Redis (Required for Job Queue)

The API uses Redis for background job processing. Start it as a Docker container:

```powershell
docker run -d --name aapsd-redis -p 6379:6379 redis:alpine
```

Verify it is running:

```powershell
docker ps
```

✅ Expected: `aapsd-redis` container shows with status `Up`.

> **After every restart**: Run `docker start aapsd-redis` to bring it back.

---

## Phase 3: Generate Kubernetes Credentials

Run the automated setup script from the project root:

```powershell
cd C:\Users\isuru\Downloads\Codex\AAPSD-Assistant
.\scripts\setup-docker-desktop-k8s.ps1
```

The script will:

1. Create a `aapsd-assistant` ServiceAccount in the cluster
2. Assign it `cluster-admin` access for testing
3. Generate a long-lived secret token
4. Print the values to paste into your `.env`

---

## Phase 4: Update the `.env` File

Open `apps/api/.env` and update the Kubernetes section with the values from the script output:

```env
# Kubernetes - Docker Desktop Built-in
# NOTE: K8S_SKIP_TLS_VERIFY=true is needed because Docker Desktop uses self-signed certs locally
K8S_SKIP_TLS_VERIFY=true
K8S_API_SERVER_URL=https://kubernetes.docker.internal:6443
K8S_TOKEN=eyJhbGciOiJSUzI1NiIs...   ← paste full token from script output
K8S_ALLOWED_NAMESPACES=default,kube-system

# Set profile to use real infrastructure
DEPLOYMENT_PROFILE=staging-remote
```

### ✅ Test 3: Governed Action Lifecycle (Full E2E)

#### Step A: Create a test deployment

In your PowerShell terminal:

```powershell
kubectl create deployment hello-nginx --image=nginx
kubectl get deployments
```

✅ Expected: `hello-nginx   1/1   Ready`

#### Step B: Request an Action in the UI

1. Go to **Governed Actions** page
2. Click **Request New Action**
3. Fill in:
   - **Action Type**: `kubernetes.deployment.restart`
   - **Deployment Name**: `hello-nginx`
   - **Namespace**: `default`
4. Click **Submit** → Status shows `pending`

#### Step C: Approve and Execute

1. Log in as an **admin** user
2. Navigate to **Governed Actions**, find the pending plan
3. Click **Approve** → Status changes to `approved`
4. Click **Execute** → Status changes to `processing` → then `completed`

#### Step D: Verify on Cluster

```powershell
kubectl rollout status deployment/hello-nginx
```

✅ Expected: `deployment "hello-nginx" successfully rolled out`

## Troubleshooting Reference

| Error                                    | Cause                        | Fix                                                                      |
| ---------------------------------------- | ---------------------------- | ------------------------------------------------------------------------ |
| `ECONNREFUSED 6379`                      | Redis not running            | Run `docker start aapsd-redis`                                           |
| `failed to connect to docker API`        | Docker Desktop not open      | Open Docker Desktop and wait 30-60s                                      |
| `unable to verify the first certificate` | Self-signed TLS cert         | Ensure `K8S_SKIP_TLS_VERIFY=true` is in `.env`                           |
| `K8S Adapter: Failed to connect`         | Wrong URL or token           | Re-run `.\scripts\setup-docker-desktop-k8s.ps1` and update `.env`        |
| `Invalid encrypted data format`          | Corrupted `.env`             | Rewrite `.env` file from scratch, ensure no null bytes                   |
| Migration fails with `null values`       | Old data conflict            | Run `npm run db:clean-dirty` then `npm run db:migrate`                   |
| `Route not found` errors                 | Old `dist/` build            | Run `npm run build` in `apps/api`                                        |
| Dashboard shows `Connecting...` stuck    | API not running / wrong port | Check the API terminal shows `Server listening at http://127.0.0.1:3000` |
